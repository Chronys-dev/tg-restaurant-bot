# Telegram Restaurant Bot

Telegram-бот для управления ресторанов. Предоставляет функционал для администраторов, директоров и сотрудников по управлению меню, рецептами, материалами, расписанием и повседневными операциями.

## Возможности

### Для администраторов
- 📋 **Управление рецептами**: добавление, редактирование и удаление рецептов
- 🏷️ **Управление тегами и соусами**: создание категорий и соусов для блюд
- 🔧 **Управление материалами**: отслеживание ингредиентов и компонентов
- 🏢 **Управление ресторанами**: конфигурация нескольких заведений
- 📢 **Объявления**: рассылка уведомлений сотрудникам
- 📊 **Поиск по рецептам**: быстрый поиск блюд по названию и тегам

### Для директоров
- 👥 **Управление персоналом**: контроль над сотрудниками и их ролями

### Для всех пользователей
- 📅 **Календарь встреч**: просмотр и управление расписанием
- ⏱️ **Завершение смены**: отслеживание рабочего времени
- 🎯 **Квизы**: тестирование знаний и обучение
- 💬 **Взаимодействия**: общение в боте
- 📝 **Материалы**: просмотр информации о материалах
- 🙏 **Благодарности**: система благодарностей и достижений
- 📨 **Утренние отчеты**: автоматические ежедневные отчеты

## Технический стек

- **Python 3.8+**
- **aiogram 3.22.0** - Telegram Bot API
- **SQLite** - база данных
- **APScheduler** - планировщик задач
- **Google Sheets API** - интеграция с Google Таблицами
- **Pydantic** - валидация данных

## Установка

### Требования
- Python 3.8 или выше
- pip (менеджер пакетов Python)

### Шаги установки

1. **Клонируйте репозиторий**
   ```bash
   git clone https://github.com/Chronys-dev/tg-restaurant-bot.git
   cd tg-restaurant-bot
   ```

2. **Создайте виртуальное окружение**
   ```bash
   python -m venv venv
   ```

3. **Активируйте виртуальное окружение**
   - Windows:
     ```bash
     venv\Scripts\activate
     ```
   - Linux/Mac:
     ```bash
     source venv/bin/activate
     ```

4. **Установите зависимости**
   ```bash
   pip install -r requirements.txt
   ```

5. **Создайте файл конфигурации `.env`**
   ```
   BOT_TOKEN=ваш_токен_бота_телеграм
   OWNER_ID=ваш_id_в_телеграме
   DATABASE_PATH=data/database.db
   REPORTS_DIR=reports
   ```

6. **Подготовьте Google Sheets интеграцию (опционально)**
   - Скачайте файл `credentials.json` из Google Cloud Console
   - Поместите его в корневую директорию проекта

## Запуск

```bash
python main.py
```

Бот начнет работать в режиме polling и будет ожидать команд в Telegram.

## Ключевые модули

### `main.py`
Главный файл приложения. Инициализирует все роутеры, базу данных и планировщик задач.

### `config.py`
Хранит конфигурацию приложения:
- BOT_TOKEN - токен Telegram бота
- OWNER_ID - ID владельца/администратора
- DATABASE_PATH - путь к базе данных
- Интеграция с Google Sheets API

### `loader.py`
Создает экземпляры диспетчера (`dp`) и бота (`bot`) для работы с Telegram API.

### `permissions.py`
Система прав доступа для различных типов пользователей (администраторы, директоры, обычные пользователи).

### `routers/`
Каждый файл содержит обработчики команд и сообщений для определенной функции:
- **admin_main.py** - основное меню администратора
- **admin_add_recipe.py** - добавление рецептов
- **admin_edit_recipe.py** - редактирование рецептов
- **admin_manage_tags.py** - управление тегами
- **user_main.py** - основное меню пользователя
- И другие...

### `services/`
Сервисы для выполнения сложных операций:
- **sheets.py** - интеграция с Google Sheets
- **scheduler.py** - планировщик повторяющихся задач
- **morning_report.py** - генерация ежедневных утренних отчетов

## Логирование

Все ошибки и события логируются в файл `bot_errors.log`. Уровень логирования установлен на `INFO`, что означает, что будут записываться все события уровня INFO и выше (WARNING, ERROR, CRITICAL).

## Работа с базой данных

База данных использует SQLite и находится в `data/database.db`. Все таблицы инициализируются автоматически при первом запуске через функцию `init_db()` в модуле `db`.

## Планировщик задач

Приложение использует APScheduler для выполнения периодических задач:
- Утренние отчеты в определенное время
- Рассылка объявлений
- Другие запланированные операции

Конфигурация планировщика находится в `services/scheduler.py`.

## Переменные окружения

Для работы бота необходимо установить следующие переменные в файле `.env`:

```
BOT_TOKEN=              # Токен бота (обязательно)
OWNER_ID=               # ID владельца бота (обязательно)
DATABASE_PATH=data/database.db      # Путь к БД (опционально)
REPORTS_DIR=reports     # Директория отчетов (опционально)
```

## Интеграция с Google Sheets

Бот поддерживает интеграцию с Google Sheets для:
- Хранения и синхронизации данных
- Создания отчетов
- Управления конфигурацией

Для использования этой функции:
1. Создайте проект в Google Cloud Console
2. Включите Google Sheets API и Google Drive API
3. Создайте сервисный аккаунт
4. Скачайте JSON-файл учетных данных как `credentials.json`

## Обработка ошибок

Бот имеет встроенную обработку исключений:
- Все ошибки логируются в `bot_errors.log`
- При критических ошибках бот пытается корректно завершить работу
- Поддерживается ручное завершение через `Ctrl+C`

## Помощь и поддержка

При возникновении проблем:
1. Проверьте файл `bot_errors.log` для ошибок
2. Убедитесь, что все переменные окружения установлены корректно
3. Проверьте, что токен бота корректен
4. Убедитесь, что вы авторизованы как администратор
